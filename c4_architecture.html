Level 2 · Containers

⚙️ How Does It Work?

Think of the system as 5 layers — like floors in a building, each with its own job.

🖥️

The Pages (What You See)

8 different screens — Dashboard, Students, Staff, Classes, Reports, Schedule, Settings, and Messages. Each role sees only what's relevant to them.

🏫 Like the front desk of a school that everyone walks into.
🧠

The Brain (DataContext)

A hidden layer that holds all information in memory while you use the app. When you change something, the screen updates instantly — no waiting.

📋 Like the school office that knows where every student file is.
📡

The Messenger (Services)

A set of helpers that talk to the database. When you add a student, the messenger carries that information to permanent storage and brings data back when needed.

📬 Like a courier that delivers letters between the office and the filing room.
💾

The Backup (localStorage)

A small copy of all data saved on your own device. If the internet disappears, you can still use the app. Changes are queued and sent automatically when you reconnect.

🗂️ Like a photocopy of important files kept in your desk drawer.
🗄️

The Database (Supabase)

The permanent, secure online storage for ALL school data. Only logged-in users can access it. New messages appear instantly thanks to a live connection.

🔒 Like the main locked filing room — the real, permanent record.

Level 3 · Database

📂 What Data Is Stored?

Think of the database as a filing room with different folders — one for each topic. There are 15 folders in total.

👨‍🎓 About Students

📁 Students

Student Records

Everything about each student.

  • Name, gender, birthday
  • Phone number
  • Enrollment date
  • Status — Active / Suspended / Dropout
📁 Attendance

Daily Attendance

Who came to school each day.

  • Student name
  • Date
  • Status: Present / Absent / Late
  • One entry per student per day
📁 Grades

Student Grades

Scores for every subject.

  • Student name
  • Subject
  • Score (0–10 scale)
  • Term (Semester 1, 2…)
📁 Enrollments

Who's in Which Class

Links students to their classes.

  • Student name
  • Class name
  • Academic year
  • One student can join many classes

👩‍🏫 About Staff

📁 Staff

Employee Records

Details of every staff member.

  • Name & role (Teacher, Guard…)
  • Subject they teach
  • Contact number
  • Date hired
  • Login password
📁 Leave Requests

Staff Permissions

When staff take time off.

  • Staff name
  • Type: Annual / Personal / Non-Personal
  • Start & end date
  • Reason given
📁 Daily Logs

Gate Entry / Exit Log

Recorded by the Guard.

  • Visitor name
  • Entry or Exit
  • Purpose of visit
  • Timestamp
📁 Incident Reports

Security Incidents

Problems reported by the Guard.

  • Title of incident
  • Description
  • Severity: Low / Medium / High
  • Date & time
📁 Room Statuses

Cleaning Updates

Updated by the Cleaner.

  • Room name
  • Status: Cleaned / Needs Attention
  • Who updated it
  • When updated

🏫 Classes, Messages & Settings

📁 Classes

Class Listings

  • Class name & level
  • Which teacher runs it
  • Schedule / timetable
📁 School Events

Calendar Events

  • Event title
  • Date
  • Type: Holiday / Exam / Meeting
  • Description
📁 Messages

Internal Messages

  • Who sent → who received
  • Text content
  • Type: Text / Leave Request / Announcement
  • Read / Unread status
📁 Settings & Config

School Configuration

  • School levels (Grade 1, 2…)
  • Subjects list
  • Timetable time slots
  • Admin password
📁 Change History

Audit Log

  • What record was changed
  • Old value vs. new value
  • Action: Added / Updated / Deleted
  • Automatic — triggered by the system

🔐 Security

Is the Data Safe?

Yes. Multiple layers of protection are built in from the ground up.

🔑

Login Required

Nobody can access any data without a valid username and password.

🎭

Role-Based Access

A Guard only sees Guard pages. A Cleaner only sees cleaning screens. Admin sees everything.

📜

Automatic Change Log

Every edit is automatically recorded — who changed what, old value, new value, and exactly when.

☁️

Cloud Hosted

Data lives on Supabase's secure servers — not just one computer. Always backed up.

📶

Works Offline

A local copy exists on your device so the app works without internet. Syncs automatically when back online.

🛡️

Row-Level Security

The database itself enforces that only authenticated users can read or write data — the app cannot bypass this.
